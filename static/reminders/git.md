# Работа с Git

Здесь коммит и изменения одно и то же.

**master** - основная ветка, содержащая все изменения входящие в последний релиз. В нее непосредственно ничего не коммитим. В нее коммиты 
попадают при слиянии в него других веток.

**debug** - ветка для тестирования, в нее тоже нельзя делать коммиты, только вливать туда другие ветки. **debug** может содержать вообще 
любые изменения, т. к. он тестовый.

------------------

# Полный процесс разработки

Итак допустим ты взял какую-то задачу и начал ее решать, и сейчас тебе надо уже изменять код. Вот, что надо перед этим сделать:

1. Перейти в ветку **master**

```bash
git checkout master
```

2. Взять все последние изменения из сервера

```bash
git pull
```

3. Ответвиться от ветки **master** в свою, где будешь писать код

```bash
git checkout -b <имя твоей ветки>
```

Делаешь коммиты в своей ветке. После того, как ты закончил задачу - пушишь свою ветку.
Дальше, тот кто делает проверку твоей работы (**reviewer**), делает вот что:

1. Берет твою ветку из сервера и переключается в нее

```bash
git fetch
git checkout <имя твоей ветки>
```

2. Если все ок, он вливает твою ветку в ветку debug

```bash
$ git checkout debug
$ git merge <имя твоей ветки>
```

3. Затем он запускает сборку ветки **debug** - мы все вместе тестим твои изменения и если все ок, то вливаем твою ветку в ветку **master** 
и твои изменения попадают в следующий релиз проекта.

```bash
git checkout master
git pull
git checkout <имя твоей ветки>
# Обязательно пересяд на master перед вливанием
# В процессе пересаживания ветки могут возникнуть конфликты, для их решения зовешь, того, кто писал код
git rebase master
git checkout master
# Обязательно включи опцию --no-ff
git merge --no-ff <имя твоей ветки>
git push
```
